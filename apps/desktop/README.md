# markdown-contract-desktop

The desktop app (D-0018 ¬ßD1): a Tauri v2 shell around a Nuxt 4 SPA, instantiated
from [template-tauri-nuxt](https://github.com/sksizer/template-tauri-nuxt) and
adapted to this workspace's Bun + moon conventions (pnpm/make/just/mise and the
template's release tooling are dropped).

## Layout

- **Project root** ‚Äî the Nuxt 4 app (bun workspace member): `nuxt.config.ts`,
  `app/` (SPA, `ssr: false`), `public/`. `nuxt generate` emits `.output/public`.
- **`src-tauri/`** ‚Äî the Rust shell crate `markdown-contract-desktop` (a member
  of the root `/Cargo.toml` workspace). `tauri.conf.json` embeds the SPA via
  `frontendDist: "../.output/public"`; the tauri CLI runs its
  `beforeDevCommand`/`beforeBuildCommand` (`bun run dev` / `bun run generate`)
  from this directory's parent ‚Äî i.e. this package root, where those scripts live.

## Features (D-0018 ¬ßD5)

A **resident vault-quality app**: the service/model layer (Vault / ScanRun /
FindingRecord / OpenerPreference over SQLite) is generated by the ontogen
pipeline (see `src-tauri/build.rs`); the resident features are hand-written
modules behind it, one responsibility per file:

- **Dashboard + vault detail** (`app/pages/`) ‚Äî vault cards (status from the
  latest run, error/warn tallies, last-scan recency, watch indicator, schedule
  chip), the add-vault form (path + name + watch toggle + optional cron), and
  per-vault detail with run history, latest findings grouped per file (kit
  `FindingRow`/`ContractGroup`), and watch/schedule editing.
- **Scan on change** (`src-tauri/src/watcher.rs`) ‚Äî a recursive `notify`
  watcher per watch-enabled vault, mirroring the Bun daemon's semantics
  (300 ms debounce, only `.md`/`.yaml`/`.yml`, `.git`/`node_modules`/dotfiles
  ignored); fires trigger `"watch"`.
- **Scheduled scans** (`src-tauri/src/scheduler.rs`) ‚Äî per-vault cron
  (`Vault.schedule`, parsed with `croner`); expressions are validated at the
  vault create/update seam (rejected as `Invalid`); fires trigger `"schedule"`.
  Watcher and scheduler both rearm off the store's change broadcast whenever a
  vault row changes.
- **Notifications** (`src-tauri/src/notifications.rs`) ‚Äî desktop toasts via
  `tauri-plugin-notification` on scan **status transitions only**
  (green‚Üífindings, findings‚Üígreen, ‚Üíerror), plus the `scan:completed` Tauri
  event the frontend listens on to refresh live (the SSE analogue).
- **System tray** (`src-tauri/src/tray.rs`) ‚Äî per-vault status rows, "Scan all
  now", "Open Markdown Contract", "Quit"; rebuilt as vaults/statuses change.
  Closing the window hides to the tray (the app is resident); the tray's Quit
  is the real exit.
- **Open in ‚Ä¶** (`src-tauri/src/openers.rs` + `app/components/OpenInMenu.vue`)
  ‚Äî [path-opener](https://github.com/sksizer/rust-path-opener)-detected apps
  (Obsidian-URI aware), merged with `OpenerPreference` rows (enabled /
  sort_order), per-vault and per-finding-file menus with launch previews.
- **Startup scan** ‚Äî every vault is swept non-blocking on app setup
  (trigger `"startup"`).

All scans converge on `src-tauri/src/scans.rs::run_scan`, which persists the
run + findings and broadcasts a completion (with the vault's previous status,
for transition detection). The template's IPC smoke test ‚Äî the `echo` command ‚Äî
stays reachable in the dashboard's dev section.

## Scan engine (D-0018 ¬ßD2)

Scans run **in-process** through `crates/markdown-contract-engine`
(`src-tauri/src/engine/`): the vault's `markdown-contract.yaml` (a
`kind: config` router, contract refs resolved relative to it ‚Äî the same
`loadConfigFile` semantics the Bun daemon uses) is compiled and the corpus
runs over the vault root, so findings carry vault-relative paths.

Vaults whose configs need the **TypeScript engine** fall back to shelling out
to an installed CLI ‚Äî `markdown-contract validate <vault> --config <file>
--format json` ‚Äî instead of failing or silently under-checking. The
`EngineRouter` delegates when:

- the config path is a `.js`/`.mjs` module, or the default YAML config is
  absent but a `markdown-contract.config.js`/`.mjs` sits in the vault root;
- loading hits a typed escape (`$ref` code escape, a `.js`/`.ts` contract
  ref, or an `mcVersion` this build doesn't support).

The CLI is looked up on `PATH` only (no `npx`/project-local resolution). If
it's missing, the run finishes as status `error` with an install pointer
(`npm install -g markdown-contract`). Any other load/run failure is an error
run with the engine's own message.

## Dev workflow

```sh
bunx moon run desktop:dev      # tauri dev: Nuxt on :1420 + native window
```

Needs a **desktop session** (a display) plus the Tauri Linux system deps
(webkit2gtk-4.1 / gtk3; the tray additionally wants libayatana-appindicator).
Override the dev port with `TAURI_DEV_PORT` (it moves the Nuxt dev server;
adjust `devUrl` in `tauri.conf.json` to match). Tray + notifications are OS
surfaces ‚Äî headlessly only their logic seams are unit-tested (transition
detection, cron next-occurrence, debounce/filter decisions, opener-preference
merging); real verification is manual on a desktop.

In a **headless container** the window can't open ‚Äî the useful subset is:

```sh
bunx moon run desktop:generate    # static SPA ‚Üí .output/public
bunx moon run desktop:typecheck   # nuxt typecheck (vue-tsc)
bunx moon run desktop:test        # bun test app (frontend unit peers)
bunx moon run desktop:check-rust  # cargo check -p markdown-contract-desktop
bunx moon run desktop:test-rust   # cargo test  -p markdown-contract-desktop
```

`tauri build` (bundling installers) is intentionally not modeled as a moon task
yet; run `bun run tauri build` manually when needed.
